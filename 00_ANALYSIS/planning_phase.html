Data model — 14 collections as implemented

Core (slate) · operational/analytics (teal). ★ new procurement collection · 🔒 append-only · ⏳ TTL 180 d · ⚡ nightly-written cache. Dashed = live reference; solid = ownership/snapshot. Mermaid source: diagrams/erd.mmd

Relationship narrative: users 1—N orders/reviews/notifications/auditLogs(actor), 1—1 carts/wishlists. categories self-reference two levels and classify products. products embed variants[] (sku, stock, priceDelta, costPriceCents, threshold, photo) and are live-referenced by carts but snapshotted into order items (name/price/cost) — products mutate, orders must not. coupons are value-referenced by code string for a human-readable audit trail. reviews carry orderId as the verified-purchase proof. purchases snapshot product and actor names so procurement history survives deletion. auditLogs use a polymorphic target (targetEntityType + id). productEvents feed the nightly engine that writes the productRecommendations cache.

Key pre-development technical decisions

  1. MERN + TypeScript workspaces monorepo — from the initial scaffold (package.json).
  2. Explicit embed-vs-reference rules — the v4 doc's decision rule ("embed when bounded, read-with-parent, not independently queried; reference otherwise") predates the code; the code follows it, including the later Purchase collection (separate, name-snapshotted).
  3. Integer cents for all money — basePriceCents, priceDeltaCents, costPriceCents, unitCostCents from the first schema.
  4. Order-item snapshotting — called "the single most important modeling decision" in v4 §3.6; implemented verbatim and later extended to purchases.
  5. Hybrid auth: short-lived JWT + rotating hashed refresh token — present since Sprint 1 (ab31544).
  6. Flat 3-role RBAC — v4 explicitly rejects a permissions collection and reverts a 4th delivery-agent role as out of BRD scope.
  7. Recommendations precomputed on a schedule, never at request time — v4 §3.10-3.11 defines the events→cache pipeline, TTL and client batching; all three exist in code.
  8. Mock-mode-first integrations — Stripe/Cloudinary/mail all run keyless for development and graded demos.
