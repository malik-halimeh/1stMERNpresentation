Corrected flow descriptions (what changed in the logic)

  • Webhook flow: now begins with a raw Buffer; in real-Stripe mode the signature check is mandatory and terminal — the "fallback to parsed body" branch no longer exists. Any description of the old fallback is obsolete.
  • Refund flow: refund is now a money-movement step that runs before the status persists; describing "refunded" as a pure status change is obsolete.
  • Stock math: all stock mutations (payment decrement, purchase addition, purchase-delete revert) are single-document atomic operations with guards; the read-modify-save description is obsolete. Purchases additionally recompute weighted-average costPriceCents inside the same update (controllers/purchase.ts:83-139).
  • Post-sign-in routing: customers → storefront home (auth-page redirects ignored); staff → admin dashboard; old "/account default" descriptions are obsolete.
  • Header search: URL-driven submit form (?search=), not a live-suggestion dropdown.
  • Secrets at boot: token signing reads the secret lazily at first use; production boot fails without it — descriptions of a module-load fallback are obsolete.
